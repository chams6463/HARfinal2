# 예제 15: 역진자 모델 기반 보행 보조 (Inverted Pendulum Control)

본 예제는 인체 보행 시 골반(trunk)을 **역진자(inverted pendulum)** 로 모델링하여, 중력 보상 토크와 PD 안정화 토크를 결합한 **모델 기반 제어(Model-Based Control)** 를 구현합니다. 단순 PD 제어(예제 14)를 넘어, 물리 모델에 기반한 고급 보조 전략을 실습합니다.

> 📖 API 레퍼런스: [H10 Control & Data](../../docs/api-reference/02-h10-control-n-data.md)

## 🎯 학습 목표 (Objective)

* **역진자 동역학**(`I * theta_ddot = M * g * L * sin(theta)`)과 보행과의 관계를 이해합니다.
* **중력 보상 토크**(`tau_gravity = M * g * L * sin(theta)`)로 역진자의 불안정성을 상쇄하는 원리를 학습합니다.
* **PD 안정화 토크**(`tau_pd = Kp * (theta_ref - theta) + Kd * (0 - theta_dot)`)로 기준 자세 부근에서 안정화하는 방법을 학습합니다.
* Homing(원점 복귀) 절차를 통해 **안전한 제어 시작** 패턴을 이해합니다.

---

## ⚙️ 동작 원리 (How it Works)

이 예제는 OFF → STANDBY → **ACTIVE**(Homing → 역진자 제어) → ERROR의 4단계 상태 머신으로 동작합니다.

### 1. Homing 절차 (`ACTIVE_PHASE_HOMING`)

ACTIVE 진입 시 먼저 모터를 **0도 위치**로 안전하게 이동시킵니다. IVector(임피던스)와 PVector(궤적)를 사용하여 부드러운 위치 제어를 수행한 후, 토크 직접 제어 모드로 전환합니다.

### 2. 역진자 실시간 제어 (`ACTIVE_PHASE_CONTROL`, 2ms 주기)

매 루프마다 다음을 수행합니다:

1.  **센서 획득:** 골반 기울기(`pelvicAngle`), 골반 각속도(`pelvicVelY`)를 읽습니다.
2.  **안전 검사:** 골반 각도가 ±30도를 초과하면 즉시 **ERROR 상태**로 전환합니다.
3.  **중력 보상:** `tau_gravity = M * g * L * sin(theta)` — 역진자의 중력 모멘트를 상쇄합니다.
4.  **PD 안정화:** `tau_pd = Kp * (theta_ref - theta) + Kd * (0 - theta_dot)` — 기준 각도 부근에서 안정화합니다.
5.  **게인 스케일링:** BTN1으로 선택한 보조 레벨(0%, 30%, 60%, 100%)을 적용합니다.
6.  **좌/우 분배:** 총 토크를 양쪽 고관절에 50:50으로 분배하여 인가합니다.

### 3. 버튼 조작

*  **BTN 1 클릭:** 보조 레벨 순환 (0.0 → 0.3 → 0.6 → 1.0 → 0.0). LED 2/3으로 현재 레벨을 표시합니다.
*  **BTN 2 클릭:** 골반 기준 각도를 현재 각도로 리셋합니다 (자세 보정).

---

## 🚀 실행 방법 (How to Use)

1.  `STM32CubeIDE`에서 본 예제 소스파일을 `user_app.c`으로 옮겨와서 빌드하고 펌웨어를 `XM10`에 업로드합니다.
2.  `KIT H10`의 전원을 켜고 `XM10`과 연결합니다.
3.  `KIT H10`의 모드를 **`ASSIST_MODE`로 변경**합니다.
4.  Homing이 진행되어 모터가 **0도 위치로 이동**하는 것을 확인합니다.
5.  Homing 완료 후, **BTN 1**을 눌러 보조 레벨을 1단계(30%)로 올립니다.
6.  슈트를 입고 몸을 앞뒤로 기울여보세요. **중력 보상 토크**가 자세를 안정화하는지 확인합니다.
7.  보조 레벨을 단계별로 높이며 보조 강도의 차이를 체감합니다.

---

## 💡 직접 해보기 (Things to Try)

* **체중/다리 길이 파라미터 변경:** `M_BODY_KG`(기본 70kg)와 `L_LEG_M`(기본 0.9m) 값을 피험자에 맞게 변경하여 중력 보상 토크의 크기가 달라지는 것을 관찰해보세요.
* **중력 보상만 vs PD만 비교:** `IP_KP_GAIN`과 `IP_KD_GAIN`을 0으로 설정하면 중력 보상만 작동합니다. 반대로 중력 보상을 제거하고 PD만 남겨보세요. 각각의 역할을 분리하여 이해할 수 있습니다.
* **기준 각도 변경:** BTN 2로 기준 각도를 다양한 자세에서 리셋하며, 다른 자세에서의 안정화 동작을 관찰해보세요.
