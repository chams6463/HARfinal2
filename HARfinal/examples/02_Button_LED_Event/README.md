# 예제 02: 버튼 이벤트와 특수 효과 (Button & LED Event)

본 예제는 단순한 상태 확인(Polling)을 넘어, 사용자의 의도를 파악하는 **이벤트(Event)** 처리 방식을 다룹니다. 버튼을 누르는 **클릭(Click)** 동작을 감지하고, 이에 맞춰 LED를 토글(Toggle)하거나 **잠깐 켜는(One-shot)** 고급 제어 기법을 구현합니다.

## 🎯 학습 목표 (Objective)

* `XM_GetButtonEvent()` API를 사용하여 **이벤트 기반(Event-Driven)** 프로그래밍을 이해합니다.
* **클릭(Click)** 이벤트를 활용하여 스위치 로직을 구현하는 방법을 학습합니다.
* 변수를 사용하여 상태(Toggle State)를 저장하고 제어하는 방법을 학습합니다.
* `XM_SetLedEffect()` API를 사용하여 별도의 타이머 코드 없이 **원샷(One-shot)** LED 효과를 구현합니다.

---

## ⚙️ 동작 원리 (How it Works)

시스템은 버튼의 상태 변화를 지속적으로 감시하다가, 특정 패턴(예: 눌렀다 뗌)이 감지되면 **이벤트 큐**에 이벤트를 저장합니다.

1.  **이벤트 감지:** 사용자가 버튼을 눌렀다 떼면 `XM_BTN_CLICK` 이벤트가 발생합니다.
2.  **이벤트 처리 (`XM_GetButtonEvent`):** 함수를 호출하여 이벤트를 읽어옵니다. 한 번 읽으면 이벤트는 사라집니다(Read-Clear).
3.  **로직 수행:**
    * **버튼 1 (토글):** 현재 LED 상태 변수(`bool`)를 반전(`!state`)시켜 켜거나 끕니다.
    * **버튼 2 (원샷):** `XM_LED_ONESHOT` 모드를 설정하여, 지정된 시간(200ms) 동안만 켜졌다가 자동으로 꺼지게 합니다.

---

## 🚀 실행 방법 (How to Use)

1.  `STM32CubeIDE`에서 본 예제 코드를 `user_app.c`에 복사하거나 프로젝트를 빌드합니다.
2.  펌웨어를 `XM10` 보드에 업로드하고 재부팅합니다.
3.  **버튼 1**을 짧게 눌렀다 뗍니다. **LED 1**이 켜진 상태로 유지됩니다. 다시 누르면 꺼집니다.
4.  **버튼 2**를 짧게 눌렀다 뗍니다. **LED 2**가 깜빡(0.2초 점등)하고 꺼집니다.

---

## 💡 직접 해보기 (Things to Try)

* **원샷 시간 늘리기:** `XM_SetLedEffect`의 마지막 인자(`period_ms`)를 `200`에서 `1000`으로 변경하여 1초 동안 켜지게 해보세요.
* **이벤트 변경:** `XM_BTN_CLICK` 대신 `XM_BTN_PRESSED`로 조건을 변경하여, 버튼을 누르는 순간 즉시 반응하도록 수정해보세요.
