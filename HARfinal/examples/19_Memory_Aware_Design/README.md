# 예제 19: 메모리 인식 설계 패턴 (Memory Aware Design)

본 예제는 MCU 환경(STM32H7, 1MB RAM)에서 `malloc/free` 없이 **정적 메모리만 사용**하는 설계 패턴을 다룹니다. 정적 링 버퍼(이동 평균 필터), 정적 풀 할당자(이벤트 로깅), `sizeof()` 기반 메모리 리포팅을 실습하며, 임베디드 양산 코드에서 왜 동적 할당을 금지하는지 체감합니다.

> 📖 API 레퍼런스: [USB Connectivity](../../docs/api-reference/05-usb-connectivity.md) · [H10 Control & Data](../../docs/api-reference/02-h10-control-n-data.md)

## 🎯 학습 목표 (Objective)

* **정적 링 버퍼(Ring Buffer):** 고정 크기 순환 버퍼로 이동 평균 필터를 구현하는 방법을 학습합니다.
* **정적 풀 할당자(Pool Allocator):** malloc 없이 고정 슬롯을 빌려주고 반환받는 패턴을 이해합니다.
* **이동 평균 필터:** 링 버퍼에 저장된 최근 N개 데이터의 산술 평균으로 노이즈를 제거하는 방법을 학습합니다.
* **이벤트 로깅:** 급격한 각도 변화를 감지하여 풀에 기록하는 패턴을 이해합니다.
* **메모리 리포팅:** `sizeof()`를 활용하여 각 구조체의 메모리 사용량을 실시간으로 확인합니다.

---

## ⚙️ 동작 원리 (How it Works)

이 예제는 **단일 상태(1-state)** TSM으로 동작합니다. 매 2ms 루프마다 센서 데이터를 처리하고, 버튼으로 리포트를 출력합니다.

### 패턴 1 — 정적 링 버퍼 (이동 평균 필터)

고정 크기(50개) 순환 배열에 매 루프마다 고관절 각도를 Push합니다. 가득 차면 가장 오래된 데이터를 자동으로 덮어씁니다. 저장된 전체 데이터의 산술 평균이 **이동 평균 필터** 출력이 됩니다.

*  **윈도우 크기:** 50 샘플 (100ms @ 2ms 주기)
*  **효과:** 고주파 노이즈 제거, 원본 대비 부드러운 곡선

### 패턴 2 — 정적 풀 할당자 (이벤트 로깅)

10개의 고정 슬롯을 미리 선언하고, 급격한 각도 변화(|delta| > 5도)가 감지되면 빈 슬롯에 타임스탬프, 각도, 토크를 기록합니다.

*  **할당:** 빈 슬롯 탐색 → 데이터 기록 (malloc 대체)
*  **해제:** BTN 2로 전체 덤프 후 반환 (free 대체)
*  **포화:** 풀이 가득 차면 LED 3 Blink 경고

### 패턴 3 — 메모리 리포팅

BTN 1을 누르면 각 구조체의 `sizeof()` 결과와 전체 정적 변수 합계를 USB CDC로 출력합니다. 전체 RAM(1MB) 대비 사용률도 함께 표시됩니다.

### USB 스트리밍

PhAI Studio에서 4채널 실시간 그래프로 확인할 수 있습니다:
*  **CH 1:** 원본 각도 (필터 전)
*  **CH 2:** 필터링된 각도 (이동 평균)
*  **CH 3:** 풀 사용 슬롯 수
*  **CH 4:** 링 버퍼 저장 요소 수

## 🚀 실행 방법 (How to Use)

1.  `STM32CubeIDE`에서 본 예제 소스파일을 `user_app.c`으로 옮겨와서 빌드하고 펌웨어를 `XM10`에 업로드합니다.
2.  USB를 PC에 연결하고 시리얼 터미널을 엽니다.
3.  `[MEM] Memory-Aware Design example started.` 메시지를 확인합니다.
4.  H10이 연결된 상태에서 관절을 빠르게 움직여 `[EVENT] Angle jump` 메시지가 출력되는지 확인합니다.
5.  **BTN 1**을 눌러 `========== MEMORY REPORT ==========` 메모리 리포트를 확인합니다.
6.  **BTN 2**를 눌러 저장된 이벤트를 전체 덤프하고 풀을 해제합니다.

## 💡 직접 해보기 (Things to Try)

* **링 버퍼 크기 변경:** `RING_BUF_SIZE`를 10(20ms)이나 200(400ms)으로 변경하여 이동 평균 필터의 응답 특성이 어떻게 달라지는지 관찰해보세요. 윈도우가 클수록 부드럽지만 지연이 커집니다.
* **풀 크기 축소:** `POOL_SIZE`를 3으로 줄인 뒤 관절을 빠르게 움직여 풀 포화(LED 3 Blink)를 유발해보세요. 포화 시 새 이벤트가 어떻게 처리되는지 확인합니다.
* **sizeof() 리포트 비교:** `DataSlot_t`의 `values` 배열 크기를 변경하거나 필드를 추가한 후 메모리 리포트를 다시 출력하여 컴파일러 패딩의 영향을 관찰해보세요.
